# Issue #1 追加検証: 厳密パーサによる再評価

## 厳密パーサの定義

| モード | 受理する出力 |
| --- | --- |
| `lenient` (これまで採用) | `[ANSWER] X` → 行頭 `X.` → 単独 `\bX\b` の順で寛容に拾う |
| `answer_tag` | `[ANSWER] X` のみ受理。それ以外は unparsable→不正解扱い |
| `letter_text` | 行頭/先頭 `X. ...` のみ受理。それ以外は unparsable→不正解扱い |

## 結果一覧

| モデル | パーサ | Acc | Correct | Unparsable |
| --- | --- | --- | --- | --- |
| baseline | lenient | 72.48% | 811/1119 | 35 |
| baseline | strict `answer_tag` | **0.00%** | 0/1119 | 1119 |
| baseline | strict `letter_text` | **0.80%** | 9/1119 | 1090 |
| SFT (letter_text) | lenient | 81.95% | 917/1119 | 0 |
| SFT (letter_text) | strict `letter_text` | **81.95%** | 917/1119 | 0 |
| SFT (answer_tag) | lenient | 82.31% | 921/1119 | 0 |
| SFT (answer_tag) | strict `answer_tag` | **82.31%** | 921/1119 | 0 |

## 考察

### 1. SFT 後はパーサ非依存

SFT を経たモデルは、訓練した形式(`X.` または `[ANSWER] X`)で**100%出力する**。厳密パーサでも寛容パーサでも全く同じ結果(917 / 921)に着地し、unparsable は 0 件。つまり SFT 後の精度数値は評価方法に依存しない。

### 2. baseline の 22.16% は本構成では再現できない

職務経歴書の baseline **22.16%** は、

- 厳密 `answer_tag` (0.00%) でも
- 厳密 `letter_text` (0.80%) でも
- 寛容 (72.48%) でも

いずれにも一致しなかった。したがって元実装は本リポと違う評価方式と推測される。可能性:
1. **両形式併用 + 追加正規表現**: `[ANSWER] X` と `X.` の両方を許容するさらに別パターン(例: 文末 `... X` や `回答は X`)を持つパーサ
2. **対数尤度ランキング** (lm-eval-harness 形式): 5択を生成させず、各選択肢の条件付き対数尤度を比較してアーグマックスを取る方式。Llama-3.2-3B は何ら追加学習なしで MCQ を 22% 程度(チャンス +1%) で当てる傾向と整合
3. **強い拒否系プロンプト**: モデルに「A/B/C/D/E のいずれか1文字だけ返す」と強要するゼロショットプロンプト

特に (2) の対数尤度ランキングが、5択チャンスレート 20% に近い 22% という数値とよく合うため最有力。

### 3. unsloth 実装の評価としては SFT 後 81.95% / 82.31% が妥当

evaluation parser 依存性が無いため、後段の Issue#2/#3 でも本値を unsloth 実装の MCQ-SFT 基準点として使える。

## 元実装(職務経歴書)との比較(更新版)

| モデル | 元実装 | unsloth 実装(本リポの厳密パーサ) | 差分 |
| --- | --- | --- | --- |
| baseline | 22.16% | 0.00%〜72.48% (定義依存) | — 直接比較不可 |
| letter_text | 85.08% | 81.95% | -3.13 pt |
| answer_tag | 85.43% | 82.31% | -3.12 pt |

SFT 後の精度差は約 -3pt で安定しており、本差分は以下が要因と推定(順):
1. 学習データ件数差(8,939 vs 9,759)
2. unsloth のプリ量子化モデル (`unsloth/llama-3.2-3b-instruct-unsloth-bnb-4bit`) を使用
3. シード/乱数性(1run のみ実施)
